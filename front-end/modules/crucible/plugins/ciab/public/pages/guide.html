1. System Overview

The CyberHub system provides isolated virtual lab environments for cybersecurity education. Each student gets their own "lane" — a private network containing:

  • Challenge VM(s) — The target system(s) students analyze (e.g., a vulnerable web server)
  • Gateway — A lightweight container that controls network access to the challenge VM
  • Attack Box (optional) — A Kali Linux VM for running security tools

Students access their VMs through Apache Guacamole, a browser-based remote desktop gateway. No VPN or special software is needed — just a web browser.

2. Account Management

Group Deployment

The admin creates groups with instructor and student accounts in bulk. When a group is deployed:

  1. Instructor and student accounts are created in the Clinic-in-a-Box system
  2. Matching Guacamole accounts are created for VM access
  3. Each account gets a unique random password (shown once at deployment — save them!)
  4. Optionally, a VM lane is deployed for each student
Tip: After group deployment, immediately download the credentials CSV. Passwords cannot be recovered later since they are hashed in the database.

Disabling Accounts

You can disable all student accounts in a group at once. Disabled accounts cannot log in. Go to the User Management tab (admin) or Access Schedule tab (instructor) to toggle.

3. Deploying Lab Environments

Pre-flight Checks

Before any deployment, the system checks:

  • Cluster resource usage (CPU, memory, storage per node)
  • Available VXLAN IDs (network isolation slots)
  • Number of concurrent deployments in progress

A confirmation dialog shows the resource impact before you commit. Deployments are blocked if resources are insufficient.

Deploying a Group

  1. Go to the Deploy Lane tab in the admin dashboard
  2. Fill in the Group Name, number of instructors and students
  3. Select a Module and Challenge from the dropdowns
  4. Check "Deploy CyberHub Lanes" if you want VMs for each student
  5. Click Deploy Lanes and confirm the resource preview
  6. Save the credentials from the results panel
Note: Lane deployment takes 30-90 seconds per student (VMs are cloned sequentially). Check the Active Lanes tab for progress.

For an individual student, deploy with # of Students set to 1.

4. Managing Active Labs

The Active Lanes tab (admin) and Lab Environments tab (instructor) show all deployed lab environments.

Deleting a Lane

Clicking Delete on a lane will:

  1. Force-stop all VMs in the lane
  2. Destroy (purge) the VMs from Proxmox
  3. Free the VXLAN ID for reuse
Warning: Lane deletion is irreversible. All data on the VMs is permanently lost.

5. Accessing Student VMs

As an instructor, you can view and connect to your students' VMs from the Lab Environments tab:

  1. Click Load IPs to fetch the current IP addresses of VMs in a lane
  2. Click Connect to open the Guacamole remote desktop in a new browser tab

You can only see lanes belonging to students in your group. To monitor a student from another group, use the Add Student search to add them to your view.

6. Access Schedules

You can restrict when students can access the system by setting an access schedule for your group:

  1. Go to the Access Schedule tab
  2. Select the active days (e.g., Mon-Fri)
  3. Set the start and end times (e.g., 8:00 AM - 5:00 PM)
  4. Choose the timezone
  5. Click Save Schedule

Students will receive a clear error message when they try to log in outside scheduled hours.

Overrides

If class time changes or you need to grant emergency access:

  • Enable Access Now — Forces all student accounts active, ignoring the schedule
  • Disable Access Now — Forces all student accounts off, ignoring the schedule
  • Clear Override — Returns to the normal schedule

7. Internet Access Control

By default, student challenge VMs are fully isolated with no internet access. You can toggle internet access per lane:

  • Admin: Toggle in the Active Lanes tab (Internet column)
  • Instructor: Toggle in the Lab Environments tab

This works by configuring iptables rules on the gateway container. When enabled, the gateway NATs traffic from the challenge VM to the internet.

Tip: Only enable internet when students need to download tools or updates. Keep it off during assessments to prevent cheating.

8. Troubleshooting

Lane stuck in "deploying"

If a lane stays in "deploying" for more than 5 minutes:

  1. Check the Proxmox web UI for stuck clone tasks
  2. The lane can be deleted and re-deployed
  3. Check the server logs for error details

Student can't connect to VM

  1. Verify the lane status is "active"
  2. Check that the student's Guacamole account exists (admin Users tab)
  3. Try clicking "Load IPs" to verify the VM has an IP address
  4. If IPs are missing, the VM may not have fully booted — wait 1-2 minutes

"Account only accessible during class hours"

The student is trying to log in outside the scheduled access window. As an instructor, you can:

  • Click "Enable Access Now" to override the schedule temporarily
  • Adjust the schedule to match the new class time

Cluster resources maxed out

If deployments are being blocked:

  1. Check the Cluster Resources bar at the top of the admin dashboard
  2. Delete unused lanes to free up VXLANs and node resources
  3. Ask the system administrator to increase resource limits
